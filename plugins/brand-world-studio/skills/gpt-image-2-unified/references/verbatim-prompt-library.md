## Format-as-prompt exemplars

These exemplars are framework-anonymized, not verbatim. Subject details are stripped; structural pattern is preserved. Captured from field-tested JSON envelope work where the structure itself acts as instruction to the reasoning model.

Source pattern: photoreal portrait preservation against beautification drift, with explicit forbidden array refusing the model's correction instinct. Canonical exemplars live in `references/json-envelope-template.md`. This section captures the structural spine for inheritance reference.

### Pattern A: Photoreal preservation envelope

Structural shape: `plan` + `search` + descriptive keys (subject, pose, environment, camera, lighting, mood, style, palette, quality, aspect_ratio) + `controlnet` block (instructional cues, not literal pipeline) + `verify` array + `negative_prompt.forbidden` array of 25-35 category-level exclusions.

Forbidden array shape:
- Anatomy correction cluster (4-5 items)
- Skin and texture artifact cluster (4-5 items)
- Lens and framing cluster (3-4 items)
- Style drift cluster (3-4 items)
- Beautification refusal cluster (4-5 items)

Use case: amateur smartphone capture authenticity, mirror-selfie portrait realism, any case where the model's safety stack would otherwise normalize the subject.

### Pattern B: Surgical edit envelope

Structural shape: `plan` (modification target named) + `subject` (modification stated explicitly) + preservation clauses on every other key + `verify` array (preservation checks) + `forbidden` array (8-12 items refusing collateral changes).

Forbidden array shape:
- Pose preservation refusals
- Framing preservation refusals
- Lighting preservation refusals
- Background preservation refusals
- Untouched-region regeneration refusals

Use case: change-one-thing edits where everything else must hold byte-identical in feel.

### Pattern C: SHOW-ME style lock envelope

Structural shape: `plan` (axis frozen vs varied stated) + `subject` (identity locked) + every non-camera key explicitly preserved + `verify` array (preservation checks per axis) + `forbidden` array (12-14 items refusing axis bleed).

Forbidden array shape:
- Wardrobe variation
- Subject identity drift
- Lighting style change
- Environment redress
- Style reinterpretation
- Mood register shift
- Filter overlay applied to new angle

Use case: angle-only variation with strict style and identity lock.

### Why these patterns hold

The model is a reasoning model. Structured forbidden arrays signal at the category level. The model unpacks each item and refuses the entire class of moves behind it. Tag-style negative prompts (single keywords) signal weakly. Cluster-style category refusals signal strongly.

Length sweet spot observed at 25-35 items for photoreal preservation, 8-12 items for surgical edits, 12-14 items for axis-lock work. Confidence: medium. Pending eval validation per subject domain.

Read `references/json-envelope-template.md` for the canonical schemas, full exemplars, and construction rules.
