# Advanced Harness Case Study

This sanitized case study shows one mature shape. It is not a default installation.

The reference system separates a global Codex home from a working workspace. It uses layered `AGENTS.md` files, project registries, reusable templates, a dated output folder, capability routing, source-owned connectors, command rules, reviewed hooks, quality gates, bounded Goals, durable loop state, and fresh-task discovery checks.

Its main lesson is architectural: identity and global policy stay small, detailed workflows live in skills and references, exact behavior moves into scripts or rules, and completion requires evidence from the live surface. The exact plugins, wording, paths, and workstreams remain user-specific.
