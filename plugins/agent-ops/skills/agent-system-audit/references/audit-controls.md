# Agent system audit controls

1. outcome: a bounded end state exists.
2. evidence: completion is tied to an inspectable surface.
3. authority: tools, systems, writes, and external actions are bounded.
4. stops: complete, pause, blocked, and escalation conditions are explicit.
5. state: durable state can survive interruption or compaction where needed.
6. tool contracts: inputs, outputs, errors, and confirmation requirements are known.
7. recovery: retries are bounded and alternate paths are documented.
8. cost: token, time, or paid-resource limits are proportionate.
9. observability: meaningful progress and failures can be inspected.
10. verification: success is checked independently of the agent's claim.
