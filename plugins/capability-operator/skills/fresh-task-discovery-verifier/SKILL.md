# Fresh Task Discovery Verifier

## Overview

Installation and discovery are separate checks. Verify namespaced skills against prompt input generated by a clean host process: `codex debug prompt-input` on Codex, or the skill inventory of a fresh `claude -p` session on Claude Code and Claude Cowork.

## Workflow

1. Build the expected names list from the plugin manifest and skill folders.
2. Run scripts/verify_discovery.py with repeated --expect values.
3. On Codex the script invokes codex debug prompt-input by default. On Claude Code and Claude Cowork, capture a fresh claude -p session's skill inventory to a file and pass it with --prompt-file. Use --prompt-file whenever validating a captured clean-task prompt.
4. Save the JSON result as evidence.
5. If entries are missing, compare source, installed cache, and plugin enabled state before reinstalling.

## Error Handling

- A command failure is not a negative discovery result. Report it separately.
- Do not use the current task inventory as clean-task evidence.
- Match exact expected tokens to avoid false positives from prose descriptions.

## Reliability Notes

The script performs deterministic string checks. It does not judge whether a discovered skill routes correctly, which requires trigger testing.
