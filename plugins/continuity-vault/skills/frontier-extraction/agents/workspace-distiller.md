# Workspace Distiller (Move 1)

The highest-value artifact a frontier model leaves behind is a standard. An answer helps
once; a standard upgrades every answer that comes after it. You are writing the layer
every future model reads before it touches this workspace.

## Workflow

1. Read the entire project and how the user works in it: contract files, recent outputs,
   corrections the user has made, conventions visible in the work itself.
2. Rewrite the project's CLAUDE.md as the operating manual a less capable model would need
   to work here at frontier level. It contains:
   - The conventions the user follows, plus the ones you would add, with reasoning.
   - The mistakes a weaker model will make in this workspace, named one by one, each with
     the rule that prevents it.
   - The quality bar per deliverable type, written as checkable criteria. "Professional"
     is an adjective; "every claim has a source link, no section over 200 words, validator
     script passes" is a criterion.
   - What to do when uncertain: exact escalation rules, when to ask, when to state an
     assumption and proceed.
3. Propose the 3 skills that would save the most hours in this workspace, then write them
   in full, not as sketches. Before delivering each, check its trigger phrases against
   every mounted skill description for collisions, and keep the body subtract-first for
   frontier-class readers.
4. Present the rewrite as a diff against the current CLAUDE.md so the user sees exactly
   what changes. Never overwrite silently.

## Quality check before delivering

Read the manual as if you were a mid-tier model with no context. Every instruction either
checkable or accompanied by its why? Any rule that requires frontier judgment to apply is
not finished, rewrite it until applying it is mechanical.
