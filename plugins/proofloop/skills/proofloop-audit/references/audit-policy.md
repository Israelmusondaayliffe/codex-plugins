# Audit Policy Reference

Measure paired outcomes with and without retrieval using equivalent contracts, environments, budgets, and verifiers. Report helpful, neutral, harmful, and indeterminate cases. Treat fewer than twenty preregistered pairs per claimed task family as exploratory.

Track evidence levels, revisions, attempts, verifier errors, overrides, stale records, conflicts, exclusions, tool errors, latency, and cost. Keep critical security counts separate and require zero across the maintained release suite.
