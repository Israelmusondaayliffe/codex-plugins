# Guardrails

The 35 controls are identified as CF-A01 through CF-D35. Each result is PASS, FAIL, BLOCKED, NOT_APPLICABLE, or UNAVAILABLE. UNAVAILABLE never counts as PASS.

Static checks cover project scope, secrets, dependency locks, file budgets, inline data, self-modification, unsafe startup writes, data separation, atomicity, concurrency, input validation, authentication, authorization, audit behavior, unsafe evaluation, and migrations. Release and lifecycle controls require explicit evidence files.
