# LinkedIn Viral Content Creator

Create LinkedIn posts that stop the scroll by combining probability-driven consensus-breaking with proven viral frameworks. Full workflow from fact validation to polished post delivery.

## Core Philosophy

Great LinkedIn content:
1. **Breaks consensus systematically** (operates at statistical edges, not mainstream middle)
2. **Stays true to user input** (no fabrication, no invented numbers, no false claims)
3. **Opens with pattern-interrupt hooks** (generated via dual methodology)
4. **Delivers value without hedging** (decisive, not wishy-washy)
5. **Sounds human, not AI** (PRISM humanization, 24-pattern AI detection, voice consistency)
6. **Shows process, not just results** (the "show work" principle)
7. **Eliminates all AI writing patterns** (em-dashes, copula avoidance, significance inflation, -ing constructions, generic conclusions, sycophancy, and more)

## Rule Zero

**Simple words win. Always.**

If you can say it plainly, say it plainly. "Use" beats "leverage." "Start" beats "commence." "Important" beats "pivotal." "Show" beats "showcase." Not trying to sound smart. Trying to sound human.

This rule sits above every other stylistic choice. When a word has a simpler equivalent that carries the same meaning, the simpler word is correct by default. No exceptions. If the complex word is the only precise option, keep it and move on. When in doubt, go simpler.

Apply this rule to everything you write in this skill, not just to text under review.

See `references/word-replacement-table.md` for the full Tier 1/2 replacement list with specific alternatives.

## CRITICAL: No-Fabrication Clause

**MANDATORY RULE:** Never invent, fabricate, or assume details not provided by the user.

**What this means:**
- If user says "tested 5 tools," never say "tested 7 tools"
- If user gives no numbers, never add numbers
- If user doesn't mention outcomes, never invent results
- If user shares vague details, ask for specifics rather than guess
- If user provides partial info, work with what's given and flag what's missing

**How to handle missing information:**
1. Extract what user provided exactly as given
2. Ask for missing details if needed for strong post
3. If user wants draft anyway, note assumptions made
4. Never fill gaps with plausible-sounding fabrications

**The iron rule:** Everything in the post must trace to user input. Period.

## Voice Profile

This is the default profile. Replace the attributes below with your own voice markers, the workflow applies whatever profile is defined here.

**Core attributes:**
- Curious, practical, collaborative (The "Buddy Collaboration Ethic")
- Direct without warmth loss
- Depth-first, clarity-second, polish-third
- No fluff, no hype, no emojis (unless explicitly requested)
- Concrete over abstract
- Questions over statements (when exploring)
- Shows work over hides process

**Writing patterns:**
- Short sentences. Punchy.
- Cinematic, poetic when appropriate
- Technical precision without jargon
- Personal without oversharing
- Uses "I tested," "I built," "I found"
- Shows uncertainty when present
- Asks for input rather than preaching
- Systematically replaces em-dashes

**Banned patterns (NEVER use):**
- Em-dashes becomes period or comma
- AI cliches: delve, leverage, unlock, journey, game-changer
- Business jargon: synergy, scalable, robust, disruptive
- Hedge language: kind of, sort of, probably, maybe
- Generic transitions: however, moreover, furthermore
- Copula avoidance: "serves as" (use "is"), "stands as" (use "is"), "boasts" (use "has")
- Significance inflation: "marking a pivotal moment," "a testament to," "reflects broader trends"
- Trailing -ing constructions: ", highlighting...", ", showcasing...", ", ensuring..."
- Negative parallelisms: "It's not just about..., it's...", "Not only...but..."
- Generic conclusions: "The future looks bright," "exciting times lie ahead"
- Sycophantic artifacts: "Great question!", "I hope this helps!"

## Complete Six-Phase Workflow

### Phase 0: Context and Fact Extraction

**FIRST PRIORITY: Extract user-provided facts**

Create mental inventory of confirmed facts. Then gather context about topic, objective, audience. If information vague or missing, ask for specifics or note what's missing. Never fabricate.

Validate before proceeding:
- No added numbers user didn't provide
- No claimed results user didn't share
- No named tools user didn't mention
- Have enough specifics for strong post

### Phase 1: Consensus Mapping

Map the "zone of indifference" (p greater than 0.90) before generating. Can't be contrarian if you don't know what's normal.

Process:
1. Identify niche/domain
2. Check `references/consensus-patterns.md` for pre-mapped consensus
3. Generate 5 most common takes on topic if unmapped
4. List standard buzzwords
5. Explain WHY consensus is stale

Output: Boring Takes list, Vocabulary Trap, Staleness Diagnosis

### Phase 2: Dual-Method Hook Generation

**Integration advantage:** Combine probability-driven contrarian angles with proven viral templates.

**Method A: Tail Sampling** (Probability-Driven)
- Generate 3-5 contrarian angles using frameworks from `references/tail-angle-templates.md`
- Economic Arbitrage, Hidden Tax, Structural Limit, False Freedom, etc.
- Assign probability scores (0.04-0.10)
- Focus on economic/structural insight

**Method B: Template Selection** (Engagement-Optimized)
- Select 3-5 fitting templates from `references/hook-templates.csv` (69 templates)
- Unique Transformation, Strong Disagreement, Quantified Strategy, etc.
- Note which align with contrarian angles

**Synthesis: Combine Both**
- Match each contrarian angle to fitting template
- Create hooks with BOTH statistical edge AND engagement structure
- Generate 5-7 hook options total

### Phase 3: PRISM Humanization

Apply PRISM Framework:
- Pattern break: Imperfection markers
- Rhythm: Vary sentence length
- Imperfection: Lowercase emphasis, fragments
- Start mid-thought: Drop into insight directly
- Meta: Self-aware commentary

Systematically replace ALL em-dashes (becomes period + capitalize OR comma + lowercase).

Load `references/prism-examples.md` for before/after examples.

### Phase 4: Body Writing

Select structure based on topic:

**Story-Driven:** Hook to Context to Discovery to Insight to Application
**Teaching-Driven:** Hook to Framework to Details to Evidence to Action
**Results-Driven:** Hook to Setup to Process to Results to Takeaway

Guidelines:
- Short paragraphs (1-3 sentences)
- One idea per paragraph
- Liberal line breaks
- Show work, not just results
- No fabricated details
- Replace all em-dashes

### Phase 5: Quality Enforcement (Updated with AI Pattern Taxonomy)

Load `references/quality-enforcement-rules.md` and enforce:

1. **Rule Zero Check**
   Every word on the Tier 1 replacement list replaced. No complex word where a simpler one works. Load `references/word-replacement-table.md`.

2. **AI Pattern Taxonomy Scan**
   Scan the full draft against the 24-pattern catalog in `references/ai-pattern-taxonomy.md`.
   Priority patterns for LinkedIn posts:
   - Copula avoidance ("serves as" for "is")
   - Significance inflation ("marking a pivotal moment")
   - Trailing -ing constructions (", highlighting...")
   - Negative parallelisms ("Not only...but...")
   - Rule of three forcing (forced triplets)
   - Generic positive conclusions ("The future looks bright")
   - Sycophantic artifacts ("Great question!")
   - Promotional language ("vibrant," "renowned," "breathtaking")

3. **Extended Pattern Scan**
   Scan against `references/extended-patterns.md` for the 12 additional patterns not in the core 24:
   - Novelty inflation ("a term nobody's naming")
   - Emotional flatline ("What surprised me most")
   - False concession structure ("While X is impressive, Y remains a challenge")
   - Rhetorical question stalls ("So why should you care?")
   - Reasoning chain artifacts ("Let me think step by step")
   - Acknowledgment loops ("You're asking about...")
   - Confidence calibration phrases ("It's worth noting," "Notably," "Interestingly")
   - "Let's" constructions ("Let's explore," "Let's dive in")
   - Template phrases ("Whether you're X or Y")
   - Vague endorsement ("worth reading," "worth your time")

4. **Negative Style Check** (remove AI tells, em-dashes, banned phrases)
5. **Cliche Scrubbing** (domain-specific overused patterns)
6. **Divergence Enforcement** (force clear stance, no hedging)
7. **Voice Validation** (voice profile markers present)
8. **Fabrication Validation** (all facts from user)
9. **Run quality scripts:**
   ```bash
   python scripts/quality_validator.py --file draft.txt --verbose
   ```

### Phase 6: Multi-Option Delivery

Present 2-3 complete post options with:
- Full hook and body
- Rationale (why this works)
- Quality score
- AI pattern scan result (clean or issues found)
- Fabrication validation
- Recommendations for when to use each

Format shows methodology, not just output.
